# Folder & File Naming Convention

## Folder Naming

All directories must follow **PascalCase with hyphens** — first letter uppercase per segment, remaining lowercase:

| ✅ Correct | ❌ Incorrect |
|---------|---------|
| `Doc/` | `doc/`、`DOC/` |
| `Firmware/` | `firmware/`、`firmWare/` |
| `Skill/` | `skill/`、`SKILL/` |
| `Esp-Build-Skill/` | `ESP-BUILD-SKILL/` |
| `Hardware/` | `hardware/` |

### Folder Rules

1. **Single word**: first letter uppercase, rest lowercase (`Doc/`, `Skill/`, `Hardware/`)
2. **Hyphen-separated words**: each segment capitalized (`Esp-Build-Skill/`, `Skill-Sync-Scripts/`)
3. **Underscore-separated words**: each segment capitalized (`My_Component/`)

## File Naming (in Doc/)

All files under `Doc/` must follow **PascalCase with hyphens**, English only:

| ✅ Correct | ❌ Incorrect |
|-----------|-------------|
| `CaPilot-PRD.md` | `CaPilot_PRD.md`、`capilot_prd.md` |
| `ESP-IDF-Pitfalls.md` | `ESP-IDF-坑点记录.md`（Chinese not allowed） |
| `Skill-Sync-Guide.md` | `skill-sync-guide.md`、`skill_sync_guide.md` |
| `Folder-Naming-Convention.md` | `folder_naming_convention.md` |

### File Rules

1. **English only** — no Chinese characters in filenames
2. **PascalCase with hyphens** — same as folder naming (`Skill-Sync-Guide.md`)
3. **Hyphen as word separator** — never underscore (`CaPilot-PRD.md` not `CaPilot_PRD.md`)
4. **Acronyms** — kept uppercase for readability (`PRD`, `ESP-IDF`)
5. **Only `Doc/` is affected** — ESP-IDF source files and components retain their original naming

### Exceptions

The following are exempt from this convention:

| Exception | Reason | Examples |
|-----------|--------|----------|
| ESP-IDF project dirs | Follow "all lowercase + hyphens" instead of PascalCase | `boot-key/`, `handheld/`, `main/`, `components/`, `spiffs/` |
| ESP-IDF components | Component name = directory name (idf convention) | `components/capilot_bsp/`, `components/capilot_usb_hid_keyboard/` |
| ESP-IDF build output | Auto-generated by toolchain | `build/`, `managed_components/`, `sdkconfig` |
| Hidden directories | Dot-prefixed meta dirs | `.git/`, `.claude/`, `.vscode/` |
| Vendor hardware material | Original files from board vendor | `Hardware/Lceda-Esp32-S3-Practical-Board/Docs/02-Chip-Manuals/` (PDFs keep original names) |
