# 【硬件交互】CaPilot：给多 AI Agent 工作流配一个“物理指挥台”

大家好，我这次参加 TRAE AI 创造力大赛的项目叫 **CaPilot**。

它不是再做一个 AI Agent，而是想做一个专门服务 **多 Agent 工作流** 的桌面硬件入口。

## 一、我想解决什么问题

现在开发者越来越常见的工作方式，是同时让多个 AI Agent 分头做事，比如写代码、查资料、跑测试、整理文档、分析报错。

但问题也很明显：**人还是只有一套键盘、鼠标和一块主屏幕**。

每次想看某个 Agent 在干嘛，都要 Alt+Tab 切窗口、翻日志、找上下文、重新聚焦。一天看下来，真正耗掉时间的往往不是“让 AI 工作”，而是“管理 AI 在哪里工作”。

我想做的，就是把这部分高频、琐碎、打断思路的管理动作，从主工作屏里拆出来。

## 二、为什么会想到做这个

我自己同时在做桌面宠物、人形机器人、博客等多个项目，已经是典型的多 Agent 使用场景。

实际体验里最痛苦的点不是 Agent 不够强，而是它们的输出散落在终端、浏览器、聊天窗口里，缺少一个统一、低打断的物理指挥界面。

刚好我手上有一块立创实战派 ESP32-S3 开发板，板载了触摸屏、麦克风阵列和 6 轴陀螺仪，天然适合把这些交互能力整合起来。所以我就想到：能不能把 **触摸板代替鼠标、语音代替键盘、体感代替快捷键** 这三件事，揉进同一个设备里，让它专门为多 Agent 工作流服务。

## 三、CaPilot 是什么

一句话说，**CaPilot 是一块 USB 即插即用的桌面小屏外设，用来查看、切换和指挥多个 AI Agent。**

它的核心形态是一个三模式设备：

- **Agent 管理模式**：一眼看到所有 Agent 的状态，比如运行中、已完成、报错、等待确认
- **触控板模式**：手指直接在屏幕上滑动，代替鼠标移动、点击和滚动
- **语音模式**：对着设备说话，把语音转成文字后发给当前选中的 Agent

未来还会结合板载陀螺仪，让“甩一下切窗口、倾斜切桌面、拍一下唤醒”这种体感交互真正变成工作流的一部分。

## 四、这个创意适合谁

我认为最核心的用户有三类：

- 同时开多个 Agent 干活的独立开发者
- 需要并行调度编码、调试、文档、测试任务的工程师
- 使用 Trae Work / Code、自定义 Agent 或多种 AI 工具协同工作的重度用户

他们有一个共同点：**双手已经被主键盘占满，但还得不断分心去管理一堆 Agent。**

## 五、会在什么场景下使用

最典型的场景是：

你正在 IDE 里写代码，但同时还让几个 Agent 分别去写模块、查资料、跑测试、总结输出。

这时候你希望：

- 抬眼就知道哪个 Agent 有结果了
- 不用切窗口就知道谁在等待你确认
- 手不用离开主工作区，就能快速切换控制对象
- 不想碰鼠标时，直接在设备上滑一下
- 不想打字时，直接说一句话把指令发出去

CaPilot 想做的，就是把这些动作都压缩成一种更自然、更低打断的桌面交互。

## 六、我认为它的价值在哪里

### 1. 效率价值

CaPilot 的目标，是把 Agent 管理相关操作的耗时降低大约 80%，把单次切换、查看、下指令的时间压到 2 秒以内。

真正的意义不只是“快一点”，而是让开发者能稳定管理更多 Agent，而不会先被上下文切换拖垮。

### 2. 品类价值

现有外设像 Stream Deck、TourBox、Magic Trackpad，各自都很强，但都不是围绕“多 Agent 指挥”这件事设计的。

CaPilot 想验证的是：**当 AI 原生工作流逐渐成为常态时，开发者是不是需要一个专门的硬件入口，来承接这些新的交互需求。**

如果这个方向成立，它不只是一个小工具，而可能是一类新的桌面生产力外设。

## 七、最后一句话总结

**CaPilot 想做的，不是再造一个 AI，而是给越来越复杂的 AI 工作流，补上一块真正顺手的“物理控制台”。**
