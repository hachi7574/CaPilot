Problem

AI 越强，管理多个 Agent 的成本反而越明显

真正卡住多 Agent 工作流的，往往不是模型能力，而是人类的输入输出通道太少。只有一套键盘、鼠标和一块主屏幕时，管理多个并发 Agent 会话很快就会变成新的瓶颈。

切换成本高

每看一个 Agent 都要 Alt+Tab、翻日志、找回上下文。单次消耗不大，但一天累积下来，真正浪费的是连续工作状态。

状态不可见

哪个 Agent 正在跑、哪个已完成、哪个在等输入，往往只能靠逐个窗口确认，关键结果很容易被日志洪流淹没。

手部动作过多

双手在主键盘和鼠标之间频繁切换，桌面上也很难再摆第二套输入设备，打断感和物理负担都很强。

效率上限被卡住

现有工具链限制了一个人能稳定调度的 Agent 数量。AI 能力还在上涨，但个人工作流的交互入口没有升级。

Product

CaPilot 是一块专门为多 Agent 工作流设计的桌面硬件

基于 ESP32-S3 开发板的现有硬件能力，CaPilot 采用小屏触控、近场语音和体感动作三种交互手段，把查看状态、切换焦点和发送指令压缩成更顺手的桌面动作。

◫
Agent 管理模式 集中展示所有 Agent 的运行状态、完成结果和是否需要人工介入，让“看全局”变成一眼动作。
✦
触控板模式 开发者不必去够鼠标，直接在屏幕上滑动、点击、滚动，就能完成高频指针操作。
◉
语音模式 把“给 Agent 下指令”从打字输入，变成说一句话即可完成，适合编码中途的快速派单。
一句话概括产品形态

一块 USB 即插即用的桌面小屏外设，让开发者双手不离主键盘，也能查看、切换并指挥多个 AI Agent。

Users

它服务的是已经进入多 Agent 工作流的人

不是所有用户都会同时开很多 Agent，但一旦进入这种工作方式，管理问题会马上浮现出来。CaPilot 主要面向已经被这类问题困住的人群。

独立开发者

一个人扛多个项目，同时让不同 Agent 负责编码、测试、文档和调研，需要低打断的统一控制入口。

多 Agent 调度工程师

同时管理多个任务流，希望随时看到哪一路在忙、哪一路出结果、哪一路需要自己确认。

AI 工作流重度用户

使用 Trae Work / Code、自定义 Agent 或多种 AI 工具协同工作，输出分散、切换频繁，最容易遇到管理瓶颈。

典型使用场景

一边在 IDE 编码，一边让多个 Agent 分头跑任务。你只需要抬眼确认状态、滑一下替代鼠标、说一句下达指令，而不用跳出主工作区。

Value

它的意义不只是更方便，而是抬高 AI 原生工作流的上限

CaPilot 想同时证明两件事：一是多 Agent 管理确实可以被硬件交互重新设计；二是 AI 原生工作方式，可能真的需要一个新的桌面入口。

效率价值

把切换、查看、确认、下指令这些动作压缩到更短路径里，目标是将 Agent 管理时间降低约 80%，让人能同时稳定驱动更多 Agent。

CaPilot 想做的，不是再造一个 AI

它想补上的，是 AI 工作流里一直被忽视的那一层交互基础设施：当一个人开始同时指挥多个 Agent 时，他需要的不再只是更强的模型，而是一块真正顺手的物理控制台。
