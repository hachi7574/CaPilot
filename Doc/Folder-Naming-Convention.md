# Folder & File Naming Convention

## Folder Naming

Repository-level product/documentation folders follow **PascalCase with hyphens**: first letter uppercase per segment, remaining letters lowercase unless the segment is a known acronym.

| ✅ Correct | ❌ Incorrect |
|---------|---------|
| `Doc/` | `doc/`、`DOC/` |
| `Firmware/` | `firmware/`、`firmWare/` |
| `Skill/` | `skill/`、`SKILL/` |
| `Esp-Build-Skill/` | `ESP-BUILD-SKILL/` |
| `Hardware/` | `hardware/` |

### Folder Rules

1. **Single word**: first letter uppercase, rest lowercase (`Doc/`, `Skill/`, `Hardware/`)
2. **Hyphen-separated words**: each segment capitalized (`Esp-Build-Skill/`, `Skill-Sync-Scripts/`)
3. **Underscore-separated words**: each segment capitalized (`My_Component/`)
4. **Tool-owned directories keep tool convention**: ESP-IDF project folders use lowercase names such as `components/`, `demos/`, `main/`, `managed_components/`, and `build/`

## File Naming

Project documentation files should follow **PascalCase with hyphens**, English only:

| ✅ Correct | ❌ Incorrect |
|-----------|-------------|
| `CaPilot-PRD.md` | `CaPilot_PRD.md`、`capilot_prd.md` |
| `ESP-IDF-Pitfalls.md` | `ESP-IDF-坑点记录.md`（Chinese not allowed） |
| `Skill-Sync-Guide.md` | `skill-sync-guide.md`、`skill_sync_guide.md` |
| `Folder-Naming-Convention.md` | `folder_naming_convention.md` |

### File Rules

1. **English only** — no Chinese characters in filenames
2. **PascalCase with hyphens** — same as folder naming (`Skill-Sync-Guide.md`)
3. **Hyphen as word separator** — never underscore (`CaPilot-PRD.md` not `CaPilot_PRD.md`)
4. **Acronyms** — kept uppercase for readability (`PRD`, `ESP-IDF`)
5. **Documentation areas are affected** — `Doc/`, `Firmware/*.md`, `Hardware/*/*.md`, and `Skill/*/SKILL.md` should follow the project convention where practical
6. **Generated/tool files are exempt** — ESP-IDF source files, lock files, config files, and component source files retain their original naming

### Exceptions

The following are exempt from this convention:

| Exception | Reason | Examples |
|-----------|--------|----------|
| ESP-IDF workspace dirs | Follow ESP-IDF conventions | `Firmware/components/`, `Firmware/demos/`, `main/`, `components/`, `spiffs/` |
| ESP-IDF components | Component name = directory name | `capilot_bsp/`, `capilot_usb_hid_keyboard/`, `capilot_usb_headset/` |
| ESP-IDF demo projects | Project names use lowercase snake_case | `audio_demo/`, `record_playback_demo/`, `usb_headset_demo/` |
| ESP-IDF build output | Auto-generated by toolchain | `build/`, `managed_components/`, `sdkconfig`, `dependencies.lock` |
| Standard project files | Conventional repository filenames | `README.md`, `SKILL.md` |
| Hidden directories | Dot-prefixed meta dirs | `.git/`, `.agents/`, `.codex/`, `.vscode/` |
| Vendor hardware material | Original files from board vendor | `Hardware/Lceda-Esp32-S3-Practical-Board/Docs/02-Chip-Manuals/` (PDFs keep original names) |
| Local reference assets | Human-readable source material may keep original language | `Hardware/参考外观设计/` |
